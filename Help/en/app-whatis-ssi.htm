What is SSI?

SSI stands for Server-Side Includes, - the mechanism allowing to create standardized templates of HTML code and insert them in the output HTML file - the file being sent to the HTTP client; this work is done automatically upon client request. This behaviour is implemented by using directives being inserted in the source HTML page. Such mechanism makes redesigning websites much easier because you have only to alter templates while the content files themselves are being left intact.

In cases "server SSI" is unavailable you can use "client SSI", when the web-designer locally processes SSI-ready content for each file of the site thus getting the copy of the source content but with include directives replaced with the corresponding files' content.

Also it is usable when no server involved in HTML deliverance at all: eg when designing HTML help files (CHM-files).

The simple example:

index.html file: 

      
    <!--#include virtual="top.ihtml" -->
    Hello World
    <!--#include virtual="bottom.ihtml" -->

    </body>

After the index.html file is processed by the SSI-server or by Client SSI Generator software <include> directives become replaced with the denoted files' content so the final index.html looks as following: 

      
    <html>
    <head><title>My website</title></head>
    <body>
    <center>
    Hello World
    </center>
    </body>
    </html>

Obviously, the simple modification of top.ihtml followed by the re-processing of the files updates all the pages mentioning it.
