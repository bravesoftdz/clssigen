Client SSI Generator

Operation

Client SSI Generator is typically installed on the web designer's workplace (the separate folder recommended), and being run on ready source-file set. The instructions for the program are being arranged as a project, which can be saved to a disk file with .cssip (client SSI project) extension. These instructions include:

  • Base HTML folder - the root source folder for the whole site. The folder is the base for all absolute links (links of kind "/file.ext")
    • .
  • Output folder - the root target folder for the generated site content. I recommend using disk mapped by using the subst command, then all the links are actually point from the root filesystem's folder, eg: 
                        subst i: C:\MyWebsite\Generated
    then you denote i:\ as an output folder.
  • Optional run-upon-completion file - you may opt to specify the file which will start automatically once generation is complete (eg, website start page). The file's name must be relative to the site's base (root) folder, eg: "music/new/myfile.html".
  • File set to process. Once base HTML folder is specified the program displays file and folder list, from which you can check those you wish to include in the final site. They might include HTML files, cascading stylesheets, (.CSS), scripts (.JS), images (.JPG, .GIF, .SWF) and all other. Notice that INCLUDE-files inselves (wearing .ihtml extensions in the examples) are not required to be included since they're needed only on generation stage.
  • SSI-extensions - the list of extensions considered to potentially contain SSI directives; these files are processed by the SSI-scanner. All other files (from checked ones) are being copied verbatim, with no content analyzing.

The program automatically creates all the folder hierarchy for target files; the only restriction is that the root output folder must exist before generation may be started.

The include directives format is as following: 

      
    <!--#include virtual="relative_path/filename.ext" -->
    <!--#include virtual="../relative_path/filename.ext" -->
    <!--#include virtual="/absolute_path/filename.ext" -->

etc.
