Client SSI Generator

Принципы работы

Client SSI Generator устанавливается на рабочем месте Web-дизайнера (желательно в отдельный каталог) и запускается при готовом наборе исходных файлов. Инструкции программе оформляются в виде проекта, который может быть сохранен в файл с расширением .cssip (client SSI project) и впоследствии открыт. Эти инструкции включают в себя:

  • Базовую папку HTML - папку, которая будет являться корневой для сайта. Из нее ведется отсчет абсолютных ссылок (вида "/file.ext")
    • .
  • Выходную папку генератора - папку, которая будет являться корневой для сгенерированного сайта. Желательно указать здесь диск, который назначен командой subst, тогда все ссылки действительно будут адресоваться из корневого каталога операционной системы, например: 
                        subst i: C:\MyWebsite\Generated
    и указать диск i:\ в качестве выходной папки.
  • Необязательный файл для запуска после генерации - после завершения формирования вы можете сразу открыть начальную страницу сайта (или любую другую). Имя файла должно быть указано относительно корневого каталога сайта, например: "music/new/myfile.html".
  • Набор файлов для обработки. После выбора базовой папки HTML основное окно программы отображает дерево файлов и папок, из которых нужно отметить те, которые должны попасть в каталог генерации. Это могут быть HTML-файлы, файлы стилей (.CSS), скрипты (.JS), изображения (.JPG, .GIF, .SWF) и всё прочее. Следует отметить, что сами INCLUDE-файлы (с расширением .ihtml в примерах выше) не требуется включать в набор выходных файлов (ведь они нужны только дизайнеру сайта).
  • SSI-расширения - расширения файлов, которые необходимо просматривать на предмет наличия в них директив include. Все прочие файлы (из отмеченных) копируются "как есть", без анализа их содержимого.

Программа сама создаёт все вложенные папки соответственно иерархии исходных файлов, однако выходная папка генератора должна существовать сразу.

Формат директив include следующий: 

      
    <!--#include virtual="относительный_путь/имя_файла.расш" -->
    <!--#include virtual="../относительный_путь/имя_файла.расш" -->
    <!--#include virtual="/абсолютный_путь/имя_файла.расш" -->

и т. д.
